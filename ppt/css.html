    Block Formatting Contexts

    块级格式化上下文, 有自己独立的布局环境和规则, 它不受外部环境影响, 也不会影响外部环境; 可以包裹内部的 float 元素, 同时不与外部 float 元素重叠.

    触发条件

    • float: 除 none 以外的值 - 浮动元素
    • position: absolute, fixed - 绝对定位元素
    • display: table-cell(table), table-caption
    • inline-block - 非块级盒子的块级容器
    • display: flex/inline-flex; 弹性盒.
    • overflow: 除了 visible 以外的值( hidden/auto/scroll )

    margin collapse

    相邻的两个box(兄弟关系或祖先关系)的外边距可能合并为一个外边距.

    合并的计算规则

    • 两外边距都是正数时, 合并结果是其中较大的值
    • 两外边距都是负数时, 合并结果是两者绝对值较大的那个
    • 两外边距一正一负时, 合并结果是两者的相加的结果

    合并条件

    • 在同一个 Block Formatting Context 中, 处于静态文档流中的块级(block/table/list-item)元素.
    • 盒子没有 padding 和 border.
    • 渲染相邻的两个元素在垂直方向上.
    • 设置 clear 属性的元素不和任何元素发生margin合并

    现象

    • BFC 内部的元素和外部元素不会发生 margin 合并.
    • 脱离文档流的元素不会发生 margin 合并 - float/absolute/fixed 元素.
    • 非 block 元素不会发生 margin 合并.

    Inline-box

    box

    containing-box/line-box/inline-box(匿名)/content-box

    line-box 在 containing-box 中上下排列, 并且它的高度取决于最高的 inline-box.

    行高(line-height)与字体尺寸(font-size)的差称为行距; 行距平分后, 分别放在 content-box 上下. 特殊: 当 font-size 大于 line-height 时, 会折叠 line-height 以保证 inline-box 高度(或 content-box 可见).

    super/sub 存在的行高会影响 line-box 高度, 细节可设置 line-height: 0; position: relative; /*+top/bottom*/ 修正;

    最后, 容器的高度是由 line-height 撑起的(不是 font-size).
